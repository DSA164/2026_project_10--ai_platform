# Mentoring — Project 11 — AI Platform

## 1. Pedagogical objective

The goal of this project is not only to produce working files. The goal is to build transferable understanding.

You should be able to explain the main concepts, justify your design choices, test your assumptions, and identify failure modes.

## 2. Concepts to understand before coding

Study and summarize the following concepts in your own words:

- plateforme locale, services LLM, RAG, workflows, agents, monitoring, déploiement.

## 3. Recommended learning loop

Use this loop for every work session:

```text
1. Choose one small objective.
2. Predict what should happen.
3. Implement the smallest useful step.
4. Run it.
5. Observe the result.
6. Explain the result in learning_log.md.
7. Only then move to the next step.
```

## 5. Mini-project deliverable

Une mini-plateforme locale avec LLM, RAG, workflow, agent contrôlé et monitoring minimal.

## 6. Validation checklist

Before considering the project complete, verify that:

- the README explains the purpose of the project;
- the main concepts are documented in `notes/`;
- each exercise has a short written observation;
- the final deliverable runs or is demonstrable;
- the limitations are explicitly listed;
- the next improvement is identified.

## 7. Questions to ask the mentor

Use questions like:

- What concept am I actually practicing here?
- What is the smallest testable version of this feature?
- What assumptions am I making?
- What failure mode should I check?
- Can I explain this without using buzzwords?

## 8. Anti-vibecoding rule

Do not paste a large generated solution directly into the project.

Prefer mentor-style interaction:

```text
I tried this.
Here is the result.
Here is what I think it means.
Where is my reasoning wrong?
```

## 9. Success criteria

Tu peux expliquer l’architecture complète, les flux de données, les points de panne et les critères de monitoring.
